# Oakland A's Roster-Era Visualization

Interactive roster-continuity views for the Oakland Athletics from 1968 through 2024. The project starts with Lahman CSVs for roster/playing-time structure, adds Baseball-Reference bWAR for player value, then renders a React + D3 web app plus static SVG/PNG exports.

## Data Sources

- Roster and playing-time source: [SABR Lahman Baseball Database](https://sabr.org/lahman-database/), CSV release. The downloader reads the official SABR Box folder and caches only the required CSVs under `data/raw/lahman/`.
- WAR source: Baseball-Reference daily WAR files, cached under `data/raw/baseball_reference/`:
  - [`war_daily_bat.txt`](https://www.baseball-reference.com/data/war_daily_bat.txt)
  - [`war_daily_pitch.txt`](https://www.baseball-reference.com/data/war_daily_pitch.txt)
- Tables used: `Batting`, `Pitching`, `Fielding`, `Appearances`, `People`, and `Teams`.
- Filter: `teamID == "OAK"` and `1968 <= yearID <= 2024`.
- Full-career overlay: all MLB `teamID + yearID` rows from 1968-2024 for players who appeared with Oakland in that range.

## Primary Metric And Core Score

The primary visualization metric is now Baseball-Reference bWAR. Batting WAR and pitching WAR are summed into `bwar` for each `playerID + yearID + teamID`. Negative WAR is preserved in the CSVs and tooltips, but visual weights use `bwar_positive = max(bwar, 0)`.

For Oakland seasons, `war_score` is normalized within that Oakland season. For full-career rows, `career_war_score` is normalized within each MLB `teamID + yearID`, so Jason Giambi's 2002 Yankees season is measured against the 2002 Yankees roster, not against Oakland players.

The Lahman `core_score` remains in the data as a swappable playing-time proxy and fallback when WAR is unavailable:

- Hitters: estimated plate appearances (`AB + BB + HBP + SH + SF`), batting games, and estimated total bases.
- Pitchers: innings pitched (`IPouts / 3`), pitching games, starts, and saves.
- Fielding/appearances: all games, games started, and fielding games.

The app uses `metric_score` / `metric_rank` as the active score. In normal builds these are bWAR-based; they fall back to the Lahman playing-time score only for a season/team with no positive WAR data.

## Run It

```powershell
npm.cmd install
npm.cmd run pipeline:download
npm.cmd run build
npm.cmd run serve:dist
```

If the raw CSVs are already cached, use:

```powershell
npm.cmd run pipeline
```

Open `http://127.0.0.1:5173/`. The normal pipeline downloads missing Baseball-Reference WAR files automatically; use `--force-war-download` to refresh that cache.

During development, `npm.cmd run dev -- --port 5173` also works in ordinary Vite environments. The checked build can always be served from `dist/` with the `serve:dist` script above.

## Analysis Checks Included

The generated analysis currently reports:

- Most roster-similar Oakland seasons by bWAR-weighted Jaccard similarity.
- Adjacent seasons with the largest core turnover.
- Bridge players between a late McGwire/Bash Brothers window (`1988-1997`) and an early Moneyball window (`2000-2004`).
- Same-community player pairs that did not overlap directly but are connected by common teammates.
- Pairs that technically overlapped but peaked in different candidate eras.

Specific examples:

- Jason Giambi and Ray Durham: no Oakland roster overlap in Lahman player-season records.
- Jason Giambi and Mark McGwire: overlap in 1995, 1996, and 1997; the generated weighted overlap is WAR-primary and regenerated by the pipeline.

## Limitations

- Baseball-Reference WAR is season/team-level here; the app still does not include transaction dates, active roster dates, postseason-only memory, or exact fan salience.
- Midseason trades are represented as season-level Oakland records, so a player traded in July still counts as overlapping that season.
- Candidate eras are algorithmic adjacent-year segments. Manual fan-facing labels should be added later as annotations rather than baked into the first pass.
- The network is limited to high-WAR Oakland players for browser performance; the full player-season table remains available in CSV.
- Non-Oakland career bars are shown in neutral gray to provide context without visually competing with Oakland seasons.
